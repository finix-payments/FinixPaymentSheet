Getting Started with PaymentSheet

What is it?

The PaymentSheet supports tokenization of credit cards and bank accounts for use with the Finix service. It is provided as a library for iOS and user code can display the payment sheet.

https://github.com/finix-payments/FinixPaymentSheet

Prerequisites

Finix Credentials

Please acquire valid finix credentials

Set up

Install the SDK

The SDK can be installed via CocoaPods. Please add the following to your Podfile:

# in your Target's Pods
pod `FinixPaymentSheet`, ~> 1.0

Install your pod dependencies.

Import the SDK

Now the SDK is ready for use in your project by importing the framework.

To access it, add the following to files that require it:

import FinixPaymentSheet

Branding

Your payment sheet can be customized by configuring PaymentInputController.Configuration. The following can be configured: - Branding - Logo iitle - Logo image - payment sheet title - button title

let branding: PaymentInputController.Branding = .init(image: UIImage(named:"YourCoLogo"), title: "YourCo")
let configuration: PaymentInputController.Configuration = .init(title: "Card Entry", branding: branding, buttonTitle: "Tokenize")

Color Theme

In addition to the branding, the sheet can be themed with different colors. By default the standard theme is used. To customize, provide a customized theme conforming to ColorThemeProtocol. Alternatively, you can take the default theme and customize colors.

let customTheme: ColorTheme = {
    var myTheme: ColorTheme = .default

    // customize button colors
    myTheme.cancelButton = .magenta
    myTheme.cancelButtonText = .red

    myTheme.tokenizeButton = .green
    myTheme.tokenizeButtonText = .black

    return myTheme
}()

Extra Customization

The form titles, placeholder text, and error messages can be customized. By default the standard set of strings is provided. To customize it you can take an instance of PaymentInputController.Localization and override specific values. Or use the initializer and provide custom values at object creation.

    var localization = PaymentInputController.Localization()
    localization.nameTitle = "名称"
    localization.addressTitle = "地址"

and then pass into the payment SDK

    var paymentSDK = PaymentAction(credentials: credentials)

    paymentSDK.localization = localization

Initialize SDK

The SDK should be initialized with an instance of FinixCredentials with your application id.

You can also specify the custom branding for your app by assigning configuration

  let credentials = FinixCredentials(applicationId: APPLICATION_ID, environment: .Sandbox)
  paymentSDK = .init(credentials: credentials)

  // Set up configuration
  paymentSDK.configuration = .init(title: "Card Entry", branding: branding, buttonTitle: "Tokenize")

Presenting

The payment sheet can be presented to tokenize two types of instruments:

  • bank account
  • credit card

Both sheets can be created from by the SDK and provide customization options including:

  • style
    • both bank and credit
    • determines the number of fields presented
  • showCancelButton
    • whether or not to show a cancel button in the UI
  • showCancelItem
    • provides a navigationItem as rightBarButtonItem to cancel presentation
  • showCountry
    • BANK-only — allow selection of country for the bank account

You must provide a PaymentActionDelegate as delegate to the controller in order to capture a sucessful tokenization.

PaymentActionDelegate

The delegate is defined as follows:

public protocol PaymentActionDelegate: AnyObject {
    func didSucceed(paymentController: PaymentInputController, instrument: TokenResponse)
    func didCancel(paymentController: PaymentInputController)
    func didFail(paymentController: PaymentInputController, error: Error)
}

On successful tokenization it will call didSucceed(paymentController:instrument:). instrument contains the API response including the id and instrument, which is the type tokenized.

If the user taps cancel via the button or the navbar item it will call didCancel(paymentController:). It’s up to your code to dismiss.

didFail(paymentController:error:) is called when the tokenization fails. error is provided to determine the reason.

Bank

 let paymentController = paymentSDK.bankPaymentSheet(showCancelButton: true,
                                                            showCancelItem: false)
        paymentController.delegate = self
        // present the configured payment controller
        paymentSDK.present(from: self, paymentSheet: paymentController)

Card

    // prepare a payment sheet with configurable cancel button, navigation cancel item, and country selection
        let paymentController = paymentSDK.paymentSheet(style: style,
                                                        showCancelButton: true,
                                                        showCancelItem: false,
                                                        showCountry: true)
        paymentController.delegate = self
        // present the configured payment controller
        paymentSDK.present(from: self, paymentSheet: paymentController)
